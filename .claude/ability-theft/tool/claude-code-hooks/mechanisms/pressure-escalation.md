# 分级干预机制

## 机制类型
状态驱动升级（PostToolUse hook + 状态文件）

## 工作原理

通过跟踪连续失败次数，在特定阈值触发不同等级的干预。每次升级附带更严格的约束要求。

```
Bash 工具调用 → PostToolUse hook 触发
    → 解析 tool_result，检测错误信号
    → 读取失败计数器
    → 判定当前等级
    → 输出对应等级的干预文本
    → 更新计数器
```

## 四级升级体系

| 连续失败次数 | 等级 | 强制动作 | 干预强度 |
|------------|------|---------|---------|
| 1 | 无干预 | — | — |
| 2 | L1 轻度提醒 | 切换本质不同的方案 | 轻度 |
| 3 | L2 中度干预 | 搜索 + 读源码 + 多个假设 | 中度 |
| 4 | L3 强制约束 | 检查清单全部完成 | 重度 |
| 5+ | L4 极限约束 | 穷尽工具或结构化退出报告 | 极限 |

## 核心实现

```bash
# 读取计数
COUNT=0
[ -f "$COUNTER_FILE" ] && COUNT=$(cat "$COUNTER_FILE" 2>/dev/null || echo "0")

if [ "$IS_ERROR" = "true" ]; then
  COUNT=$((COUNT + 1))
  echo "$COUNT" > "$COUNTER_FILE"
else
  if [ "$COUNT" -gt 0 ]; then
    echo "0" > "$COUNTER_FILE"  # 成功重置
  fi
  exit 0
fi

# 等级路由
if [ "$COUNT" -lt 2 ]; then exit 0; fi    # 第1次失败：不干预

if [ "$COUNT" -eq 2 ]; then
  # L1 输出
  cat << EOF
[Intervention L1 ${STYLE_ICON} — Consecutive Failure Detected]
> ${INTERVENTION_L1}
...
EOF
elif [ "$COUNT" -eq 3 ]; then
  # L2 输出
  ...
elif [ "$COUNT" -eq 4 ]; then
  # L3 输出
  ...
else
  # L4 输出（5+ 次）
  ...
fi
```

## 升级内容结构

每个等级的输出包含以下部分：

```
[等级标签 + 图标 + 触发原因]
> 干预提示文本（1-3 句）

强制动作清单：
- [ ] 具体动作 1
- [ ] 具体动作 2
...

方法论切换指令（L2+）
当前风格信息
```

## 失败模式检测与切换

L2 以上强制切换干预风格，禁止继续使用当前失败的方法：

| 失败模式 | 检测信号 | 强制动作 |
|---------|---------|---------|
| 反复尝试相同方案 | 相似命令重复 | 强制切换方法论，禁止重试相同路径 |
| 直接放弃 | "can't solve" | 拆分为子问题，逐个击破 |
| 表面修复 | 只改症状不改根因 | 强制根因分析，出示证据链 |
| 不查就猜 | 不读代码就断言 | 强制读源码，禁止凭猜测修改 |

## 设计要点

1. **成功重置**：任何一次成功调用都会将计数器归零
2. **首次不干预**：第一次失败不触发，给自然重试空间
3. **等级递增**：每次升级都是更严格的约束，不会降级
4. **风格感知**：干预文本来自 `style-helper.sh`，可配置
5. **跨工具无关**：只跟踪 Bash 工具的失败，其他工具不影响
