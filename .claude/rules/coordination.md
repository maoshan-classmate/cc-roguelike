# 协调规则（与 PUA 角色并行生效）

## 委派规则

| 从 | 可以委派给 |
|---|----------|
| P10 (CTO) | P9（战略方向 → 任务定义） |
| P9 (Tech Lead) | P8（Task Prompt → 独立执行） |
| P8 (独立一面) | P7（子任务拆解 → 方案驱动执行） |
| P8 | 同级 P8（横向咨询，不做约束性决策） |

**冲突升级**：P8 冲突 → P9 裁决 → P10 仲裁

## 5 条核心规则

1. **垂直委派** — P10/P9/P8/P7 不跳级委派复杂决策。简单信息查询可跨级。
2. **横向咨询** — 同级 P8 可互相咨询，但不在非自己负责的领域做约束性决策。
3. **冲突解决** — 升级到共同上级。无共同上级则升级到 P10。
4. **变更传播** — 跨领域变更由 P9 协调传播，涉及方必须收到通知。
5. **禁止单方面跨领域修改** — P8/P7 不修改非自己负责的文件域，除非显式委派。

## 跨领域变更通知

修改以下文件域时，必须通知相关方：
- `server/` 变更 → 通知前端 P8（`src/` 影响）
- `src/config/` 变更 → 通知服务端 P8 + sprite-audit
- sprite 3 文件任一变更 → 触发 sprite-audit 审查
- `docs/gdd/` 变更 → 运行 `/propagate-design-change` 检查影响

## 反模式

- 跳级委派复杂决策
- 同级 P8 在非自己领域做约束性决策
- P7/P8 单方面修改非负责文件域
- 跨领域变更不通知相关方
