# 上下文管理

**核心原则**：文件是记忆，不是对话。对话会被压缩或丢失，文件持久存在。

## Session State 文件

`.claude/session-state/active.md` 是活检查点。每个重要里程碑后更新：
- 设计决策通过后
- 架构变更后
- 实现里程碑完成后
- 测试结果获取后

状态文件包含：当前任务、进度清单、关键决策、正在修改的文件、未解决问题。

## 增量文件写入

创建多章节文档时：
1. 立即创建骨架（所有章节标题，空 body）
2. 每个章节在对话中讨论和起草
3. 每个章节通过后立即写入文件
4. 更新 session state

这保持上下文窗口只持有**当前章节**的讨论（~3-5k tokens），而非整个文档历史（~30-50k tokens）。

## 主动压缩

- **60-70% 上下文用量时主动压缩**，不要等到极限
- **自然压缩点**：写入文件后、提交后、完成任务后、开始新话题前
- **`/clear`** 用于不相关任务之间

## 压缩保留信息

压缩摘要必须包含：
- session state 文件引用（`.claude/session-state/active.md`）
- 本会话修改的文件列表及用途
- 架构决策及理由
- 活跃 TODO 任务及状态
- agent 调用及结果
- 测试结果
- 未解决的阻塞问题
- 当前任务和步骤

## 会话恢复

压缩后或新会话：
1. 读取 `.claude/session-state/active.md`
2. 读取正在工作的文件
3. 从下一个未完成章节或任务继续
