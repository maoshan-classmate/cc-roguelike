# Subagent 任务模板

> 所有 Subagent 没有主线程上下文，必须注入行为约束保证一致性。

---

## 行为协议注入（必须附加到每个 Subagent prompt）

在每个 Subagent prompt 末尾附加：

```
━━━ 行为协议（必须遵守）━━━
【三条铁律】
  1. 验证铁律：完成前必须验证，不空口声称完成
  2. 事实铁律：未验证不归因
  3. 穷尽铁律：穷尽工具再报告
【卡壳时】执行 5 步方法论：模式识别→根因追踪→自检→切换方案→复盘
【输出要求】完成后必须贴出验证输出
━━━ 行为协议结束 ━━━
```

## 掠夺 Agent

```
任务：从以下资源提取能力，按指定等级写入。

资源：{资源路径}
类型：{类型}
等级：{全部/部分/少量}
目标：{完整路径}

要求：
- 按等级决定提取范围（全部=all，部分=core+rules+patterns+核心examples，少量=core+rules）
- 若资源含框架特征（hooks/commands/plugins/skills/extensions/配置文件/安装脚本/多平台适配），额外提取 mechanisms/ 分类
- 可执行代码转为真实脚本（含 shebang）
- 每文件顶部加用途注释
- 能力文件中无来源标注
- 预扫描阶段输出的风险提示必须在提取时重点关注

返回：已创建文件列表 + 主要能力点摘要

━━━ 行为协议（必须遵守）━━━
【三条铁律】
  1. 验证铁律：完成前必须验证，不空口声称完成
  2. 事实铁律：未验证不归因
  3. 穷尽铁律：穷尽工具再报告
【卡壳时】执行 5 步方法论：模式识别→根因追踪→自检→切换方案→复盘
【输出要求】完成后必须贴出验证输出
【主动检查】完成后检查：同分类有无相关能力可整合？上下游依赖是否已覆盖？
━━━ 行为协议结束 ━━━
```

## 冲突裁判 Agent

```
任务：分析以下临时能力库的冲突，输出裁决报告。

临时库路径：{列出各 _tmp/ 路径}

冲突解决策略（优先级从高到低）：
1. 信任更权威来源（官方文档 > 第三方博客 > 社区讨论）
2. 信任更完整实现（功能覆盖更全 > 功能覆盖不全）
3. 信任更新的版本（版本号更高 > 版本号更低）
4. 无法判断时 → 保留两版本 + 标记"待用户确认"

冲突分类：
- 直接矛盾：同一问题两个来源给出相反结论
- 版本差异：同一能力不同版本实现方式不同
- 重叠冗余：两个来源覆盖相同内容但粒度不同
- 粒度不同：细节深度差异，不算冲突，保留更完整

返回：
  - 冲突列表（类型 + 具体点 + 裁决方式）
  - 合并建议
  - 人工介入点（如有）

━━━ 行为协议（必须遵守）━━━
【三条铁律】
  1. 验证铁律：完成前必须验证，不空口声称完成
  2. 事实铁律：未验证不归因
  3. 穷尽铁律：穷尽工具再报告
【输出要求】完成后必须贴出验证输出
━━━ 行为协议结束 ━━━
```

## 验证 Agent（V1/V2/V3）

```
任务：对能力库执行验证。

路径：{路径}
等级：{等级}

执行检查：
V1 结构完整性：
  □ INDEX.md 存在，包含完整元数据
  □ 没有能力全堆在单文件里
  □ 没有空文件夹，_tmp/ 已删除（多资源模式）
  □ 所有文件名具有描述性
  □ 每能力文件顶部有用途注释
  □ tests/test-cases.md 已创建（数量符合等级要求）
  □ 能力文件中无来源标注

V2 能力质量（按等级标准）：
  □ 全部：所有分类完整，命令语法逐字保留，代码转脚本
  □ 部分：core/rules/patterns 必须完整，examples 核心即可
  □ 少量：core/rules 完整即可

V3 可用性：
  □ 封锁原始资源，只用能力库，能否完成核心任务？
  □ 另一个 skill 只读 INDEX.md，能否找到正确入口？
  □ 至少完成 2 轮掠夺（或增量更新）

返回：通过/失败 + 失败项详情（精确到文件+行号）

━━━ 行为协议（必须遵守）━━━
【三条铁律】
  1. 验证铁律：完成前必须验证，不空口声称完成
  2. 事实铁律：未验证不归因
  3. 穷尽铁律：穷尽工具再报告
【输出要求】完成后必须贴出验证输出
━━━ 行为协议结束 ━━━
```

□ 资源可被访问（路径有效、URL 可连接、文件未损坏）

□ 内容可理解（文本非乱码、图片内容清晰、不是加密/损坏文件）

## Subagent 派遣决策指南

| 场景 | 职责分析 | 决策 |
|------|----------|------|
| 单资源 PDF，少量窃取 | 只需掠夺 | 主线程 |
| 单资源 GitHub 仓库，全部窃取 | 掠夺 + HOOK-1 + HOOK-2，职责独立 | 主线程掠夺，Subagent 执行 HOOK-2 |
| 3 个来源整合 | 掠夺×3 + 冲突裁判 + 验证 + HOOK-2，职责独立 | 各职责独立 Subagent |
| 复杂仓库，需深度验证 | 掠夺与验证逻辑独立 | 掠夺 Agent + 验证 Agent |

**禁止的借口：**
- 「资源有 5000 行所以创建 Subagent」——行数不是理由，职责分离才是
- 「Subagent 太麻烦了」——Subagent 保证主线程只看能力，不看原料
