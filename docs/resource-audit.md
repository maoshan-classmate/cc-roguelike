## 六、角色精灵 Bug 分析

### 根因

`src/config/characters.ts` 假设每个角色有 4 个方向精灵：

```typescript
warrior: { front: 0, back: 1, left: 2, right: 3 }  // ← 索引 2,3 是空白/微小物品！
```

**像素分析证实**：
- 索引 2：**空白**（填充率 <5%，不在分析结果中）
- 索引 3：填充率 10.9%，主色 (64,64,64)，是一个微小物品，**不是角色**
- 索引 4,5：**空白**
- 索引 6-9：橙色装备（填充率 30-40%），**不是角色**

### 实际角色布局

roguelikeChar 中角色按 **2 个一组** 排列（正面+背面），不是 4 个一组：

| 角色组 | 索引 | 方向 |
|--------|------|------|
| 战士（基础） | 0=正面, 1=背面 | 无左右 |
| 战士（装备A） | 54, 55 | 正面+背面 |
| 战士（装备B） | 108, 109 | 正面+背面 |
| 绿色角色 | 162, 163 | 正面+背面 |
| 橙色角色 | 270, — | 仅正面 |
| 青色角色 | —, 271 | 仅背面 |
| ... | ... | ... |

### 修复方案

1. **左右朝向用镜像**：当角度指向左右时，水平翻转正面/背面精灵
2. **修正 characters.ts 中的精灵索引映射**
